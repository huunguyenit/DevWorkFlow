Credit/Debit Memo

Credit/Debit Memo is used to record arising adjustment of payable amount such as purchase discount, settlement discount and other reasons.

Special feature of this voucher is that payable amount can be reduced directly in invoice or by supplier. In case of reduction by supplier with payment control detailed by invoice, it is required to allocate to invoices.

Field descriptions - format view

Cust. ID

Customer ID, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units. In case transactions which are not related to debt control, this information is used to control internal objects as production line, shop floor…

Receiver

Person receives cash; extracted from Supplier List (“Partner” field) correlative with Supplier ID entered or free entered by user. This field is declared to use or not in “Data Enter Screen Declaration”;

Cr. Account

Credit account used to reflect arising business activities, extracted from Chart of Accounts.

Trans. Type

1 - Adjustment Detailed by Invoice, 2 - Adjustment Detailed by Customer, 3 - Adjustment for many Customers, 5- Adjustment Detailed by Customer (FC), 9 - Other. These types are used to define adjustment method: directly decrease to invoice, decrease receivables of a supplier or decrease receivables of many suppliers at the same time. Default is type 2; however, it can be edited in “Voucher Information” function.

Voucher No.

It is automatically increased 1 when a new voucher entered but editable. When you save debit memo, the system will check the voucher number duplicate. You can define to allow the voucher number duplicate during a day, month, quarter, year or in the whole database.

Voucher Date

The date the voucher is created, reference information only. Program will use GL Date in calculation.

GL Date

The date the voucher is posted; is default as created date in calculation.

Ex. Rate

If currency is GL currency, exchange rate will be 1 and this field will not be updated. If exchange rate is changed, program will recalculate amount related to foreign currency and GL currency.

Field descriptions - “Details” tab – decrease details in invoice

Inv. No.

Number of invoices which are not paid or partially paid to supplier, extracted from list of invoices without full paid.

Inv. Date

The date the invoice is created.

FC Code

Code of transaction currency in invoice

Dr. Acct.

Debit account in transaction decrease payable to supplier. Dr. Account is automatically updated based on payable account used to refer to transaction when having invoice. Dr. Account in this case cannot be entered but extracted from invoice.

Invoice Amt.

Amount in transaction currency in invoice; automatically updated.

Paid

Amount paid to invoice; automatically updated (converted to transaction currency in invoice).

Payable

Amount will be paid to invoice (in transaction currency); automatically updated.

FC Payment

Payment in transaction currency of debit memo

Converted Payment

If transaction currency in debit memo is different from that in invoice, Converted Payment field will be updated to have amount converted to transaction currency in invoice. Otherwise; this field will not be updated.

Description

Transaction reasons/ memo; posted to GL as Description field.

Payment

Payable amount in GL currency. If transaction currency is the same as GL currency, this field will not be displayed. This field value equals FC amount multiplied by exchange rate, but editable.

FC Amount

Some common operations

Calculate instant exchange rate

Like Disbursement, payable adjustment voucher is allowed to calculate difference between book rate and real exchange rate. Press F5 to calculate exchange rate of FC account related to entry in voucher.

Edit book rate

In case automatic rate is not used to post to GL, it can be changed by option “Edit Book Rate”. These vouchers will not be updated when calculating monthly average exchange rate.
See common guides of data entry for more reference of other operations.
