# 08 — Update Docs

---

**Prompt:**

Tôi vừa thay đổi: **<mô tả thay đổi kiến trúc/API/behavior>**

Yêu cầu:

1. Xác định tài liệu nào trong bộ (`00-VISION.md` .. `10-DEVWORKFLOW_BOOK.md`, `ADR-000x`) cần cập nhật.
2. Giữ nguyên văn phong, cấu trúc heading (`#`, `---`) và ngôn ngữ (tiếng Việt) như tài liệu gốc.
3. Nếu thay đổi là quyết định kiến trúc mới (không chỉ chỉnh sửa nhỏ), đề xuất tạo ADR mới
   thay vì sửa ADR cũ (dùng `09-GENERATE_ADR.md`), và thêm liên kết vào "Related Documents"
   của các ADR liên quan.
4. Cập nhật mục "Related Documents" hai chiều nếu tài liệu mới tham chiếu tài liệu cũ.
5. Không thay đổi các phần "Status: Accepted" của ADR đã duyệt trừ khi tôi xác nhận đây là
   một "Future Review" hợp lệ theo điều kiện ghi trong chính ADR đó.
