Credit Memo

Credit Memo is used to record arising adjustment of receivable amount such as sales discount, settlement discount and other reasons.
Special feature of this voucher is that receivable amount can be reduced directly in invoice or by customer. In case of reduction by customer with payment control detailed by invoice, it is required to allocate to invoices.

“Credit Memo” entry screen

Field descriptions - format view

Cust. ID

Customer ID, extracted from Customer/Supplier List, is usually used in debt control in case of Issue related to debt transactions as lending items from other units. In case transactions which are not related to debt control, this information is used to control internal objects as production line, shop floor…

Payer

Person pays cash; extracted from Customer List (“Partner” field) correlative with Customer ID. Information of this field is defined in “Voucher Enter Screen Declaration”

Dr. Account

Debit account reflects transaction, extracted from Chart of Accounts.

Trans. Type

1 – Receipt detailed from invoice, 2 – Other receipt: of 1 customer, 3 – Other receipt: of many customers.
These types are used to define adjustment method: directly decrease to invoice, decrease only to receivable account of customer or decrease receivable of many customers at the same time. Default is type 2; however, it can be changed in “Voucher Enter Screen Declaration”.

Voucher No.

It is automatically increased by one value when a new voucher entered but editable. Program will check the number coincidence daily, monthly, quarterly, yearly or all database.

Voucher Date

The date the voucher created; reference information only. Program will use GL date to calculate.

GL Date

The date the voucher is posted; used in calculating and is default as created date.

Ex. Rate

If transaction currency is the same as GL currency, exchange rate will be 1 and will not be entered. If exchange rate is changed, program will recalculate value related to foreign currency and GL currency.

Field descriptions - “Details” tab – decrease details in invoice

Inv. No.

Number of invoices which are not paid or partially paid by contract, extracted from invoice list.

Inv. Date

Date created invoice

FC Code

Code of transaction currency in invoice

Cr. Acct.

Cr. Account in transaction decrease receivable from customer. Cr. Account is automatically updated based on receivable account used to reflect transaction when having invoice. Cr. Account is extracted from invoice.

Invoice Amt.

Amount in transaction currency in invoice; automatically updated.

Paid

Amount received from invoice; automatically (converted to transaction currency in invoice). .

Payable

Amount has been receivable from invoice (in transaction currency); automatically updated basing on amount paid to invoice (in invoice currency).

[FC] Payment

Amount remaining in transaction currency of credit memo. .

Converted Payment

If transaction currency in credit memo is different from that in invoice, Converted Payment field will be updated to have amount converted to transaction currency in invoice. Otherwise, this field will not be updated.

Description

Transaction reasons/ memo; posted to GL as Description field.

Payment

Amount receivable in GL currency. If transaction currency is the same as GL currency, this field will not be displayed. This field value is equal to FC amount multiplied by exchange rate and editable to make it round.

Field descriptions - “Details” tab – decrease and later allocate

Cr. Account

Code of Cr. Account in transaction, extracted from Chart of Accounts

FC Amount

Amount in transaction currency. If there is change of FC amount, amount will be recalculated=FC amount exchange rate. Field width is 12 characters with 2 decimal number characters. Negative number is allowed.

Amount

Amount in GL currency. This field will not be displayed if transaction currency is GL currency. Its value is equal to FC amount multiplied by exchange rate but editable to make it round. Field width is 12 characters with 2 decimal number characters. Negative number is allowed.

Description

Description of transaction‘s reasons, posted to GL as Description field.
